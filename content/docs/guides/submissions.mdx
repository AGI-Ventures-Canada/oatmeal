Participants create, update, and upload submissions through the web interface at `/e/{slug}`. As an organizer, you can browse all submissions via the API or CLI.

## How Participants Submit

Participants register for your hackathon, build their project, and submit through the event page. They can update their submission at any time before the deadline. No API setup is needed on your end — the web interface handles the full flow including screenshot uploads.

### Submission Fields

| Field | Constraints |
|-------|-------------|
| `title` | 1-100 characters |
| `description` | 1-280 characters |
| `githubUrl` | Must be a github.com URL |
| `liveAppUrl` | Optional — link to a deployed demo |

## Browsing All Submissions

View all submissions in the gallery (public, no auth required):

```bash title="List all submissions"
curl -s "$BASE_URL/api/public/hackathons/$SLUG/submissions" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon browse submissions $SLUG --json
```
</Callout>

## Checking a Submission

Individual participants can check their own submission status at any time through the web interface at `/e/{slug}`. This is a participant-only action and is not available via the API.

## Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| `not_registered` | Haven't registered yet | Register first via `/register` |
| `submissions_closed` | Submission window closed | Check hackathon dates |
| `already_submitted` | Already have a submission | Use PATCH to update instead |
| `invalid_github_url` | URL isn't from github.com | Use a github.com repository URL |

## Best Practices

1. **Submit early** — Don't wait until the last minute. Submit a draft and update it as you go
2. **Add a screenshot** — Submissions with screenshots stand out in the gallery and during judging
3. **Include a live demo** — Judges appreciate being able to try your project
4. **Keep the description concise** — 280 characters max, so focus on what makes your project unique
5. **Update your GitHub repo** — Make sure the repo is public and the README explains how to run the project
