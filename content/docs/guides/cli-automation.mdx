The Oatmeal CLI is designed for automation. Every command supports `--json` output and `--yes` to skip prompts, and you can authenticate via environment variables instead of interactive login.

## Authentication in CI

Use the `OATMEAL_API_KEY` environment variable — no `oatmeal login` needed:

<CodeBlock language="bash" showLineNumbers code={"export OATMEAL_API_KEY=sk_live_your_key_here\noatmeal hackathons list --json"}>

Destructive commands (delete, publish, unpublish) require confirmation by default. Use `--yes` to skip:

## Tips

- **Always use `--json`** in scripts — table output can change between versions
- **Use `--yes`** for all destructive commands in CI
- **Set `NO_COLOR=1`** to disable ANSI colors in log output
- **Non-TTY detection**: The CLI automatically uses non-interactive mode when `stdout` is not a terminal
- **Exit codes**: Commands return `0` on success and `1` on failure, so `set -e` works correctly in shell scripts
