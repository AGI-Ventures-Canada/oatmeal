The Oatmeal CLI is designed for automation. Every command supports `--json` output and `--yes` to skip prompts, and you can authenticate via environment variables instead of interactive login.

## Authentication in CI

Use the `OATMEAL_API_KEY` environment variable — no `oatmeal login` needed:

```bash title="Environment variable auth"
export OATMEAL_API_KEY=sk_live_your_key_here
oatmeal hackathons list --json
```

In CI platforms, set `OATMEAL_API_KEY` as a secret:

```yaml title=".github/workflows/example.yml"
# GitHub Actions
env:
  OATMEAL_API_KEY: ${{ secrets.OATMEAL_API_KEY }}

steps:
  - run: npx @oatmeal/cli hackathons list --json
```

To point at a local dev server instead of production:

```bash title="Local development"
export OATMEAL_BASE_URL=http://localhost:3000
```

## JSON Output

Every command supports `--json` for machine-readable output:

```bash title="JSON examples"
# Pipe to jq for filtering
oatmeal hackathons list --json | jq '.[0].id'

# Save to file
oatmeal results get <hackathon> --json > results.json

# Use in shell scripts
HACK_ID=$(oatmeal hackathons list --json | jq -r '.[0].id')
oatmeal prizes list $HACK_ID --json
```

## Skip Confirmation Prompts

Destructive commands (delete, publish, unpublish) require confirmation by default. Use `--yes` to skip:

```bash title="Non-interactive mode"
oatmeal hackathons delete <id> --yes
oatmeal results publish <hackathon> --yes
oatmeal prizes unassign <hackathon> <prize-id> <submission-id> --yes
```

## Example: Automated Hackathon Setup

Create a script that sets up a hackathon with standard criteria and prizes:

```bash title="setup-hackathon.sh"
#!/bin/bash
set -euo pipefail

SLUG="$1"
NAME="$2"

# Create hackathon
oatmeal hackathons create --name "$NAME" --slug "$SLUG"

# Add standard criteria
oatmeal judging criteria create "$SLUG" \
  --name "Innovation" --max-score 10 --weight 30
oatmeal judging criteria create "$SLUG" \
  --name "Technical Execution" --max-score 10 --weight 40
oatmeal judging criteria create "$SLUG" \
  --name "Presentation" --max-score 10 --weight 30

# Add standard prizes
oatmeal prizes create "$SLUG" \
  --name "Grand Prize" --type monetary --value 5000
oatmeal prizes create "$SLUG" \
  --name "Runner Up" --type monetary --value 2500
oatmeal prizes create "$SLUG" \
  --name "Best Design" --type custom --value "Design toolkit"

echo "Hackathon '$NAME' created with criteria and prizes."
```

Run it:

```bash title="Terminal"
chmod +x setup-hackathon.sh
./setup-hackathon.sh spring-hack-2026 "Spring Hack 2026"
```

## Example: Post-Judging Workflow

Automate the results flow after judging completes:

```bash title="finalize-results.sh"
#!/bin/bash
set -euo pipefail

HACKATHON="$1"

# Calculate results
oatmeal results calculate "$HACKATHON" --yes

# Export results for review
oatmeal results get "$HACKATHON" --json > "results-$HACKATHON.json"

echo "Results calculated and saved to results-$HACKATHON.json"
echo "Review the file, then run:"
echo "  oatmeal results publish $HACKATHON"
```

## Example: GitHub Actions — Publish Results on Schedule

```yaml title=".github/workflows/check-results.yml"
name: Check & Publish Results

on:
  schedule:
    - cron: '0 */6 * * *'  # Every 6 hours
  workflow_dispatch:

jobs:
  check-results:
    runs-on: ubuntu-latest
    env:
      OATMEAL_API_KEY: ${{ secrets.OATMEAL_API_KEY }}
    steps:
      - name: Check judging progress
        run: |
          PROGRESS=$(npx @oatmeal/cli judging judges list ${{ vars.HACKATHON_ID }} --json)
          echo "$PROGRESS" | jq '.[] | {name, completed, total}'

      - name: Calculate results if all scored
        run: |
          npx @oatmeal/cli results calculate ${{ vars.HACKATHON_ID }} --yes || true
```

## Example: Webhook + CLI Integration

Use webhooks to trigger CLI commands when events occur:

```bash title="Webhook-triggered automation"
# Set up a webhook for submission events
oatmeal webhooks create \
  --url https://your-server.com/hooks/oatmeal \
  --events submission.submitted

# In your webhook handler, trigger auto-assign when all submissions are in
# (pseudocode)
curl -X POST your-server.com/hooks/oatmeal
# → handler runs: oatmeal judging auto-assign $HACKATHON --per-judge 5
```

## Tips

- **Always use `--json`** in scripts — table output can change between versions
- **Use `--yes`** for all destructive commands in CI
- **Set `NO_COLOR=1`** to disable ANSI colors in log output
- **Non-TTY detection**: The CLI automatically uses non-interactive mode when `stdout` is not a terminal
- **Exit codes**: Commands return `0` on success and `1` on failure, so `set -e` works correctly in shell scripts
