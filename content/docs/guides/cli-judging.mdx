This guide covers the complete judging workflow — defining criteria, inviting judges, distributing submissions, and monitoring scoring progress.

## Define Judging Criteria

Criteria define how judges score submissions. Each criterion has a name, maximum score, and weight (percentage of the final score):

<CodeBlock language="bash" showLineNumbers code={"oatmeal judging criteria create <hackathon> \\\n  --name \"Innovation\" \\\n  --description \"Novelty and creativity of the solution\" \\\n  --max-score 10 \\\n  --weight 30\n\noatmeal judging criteria create <hackathon> \\\n  --name \"Technical Execution\" \\\n  --description \"Code quality, architecture, and completeness\" \\\n  --max-score 10 \\\n  --weight 40\n\noatmeal judging criteria create <hackathon> \\\n  --name \"Presentation\" \\\n  --description \"Demo quality and clarity of explanation\" \\\n  --max-score 10 \\\n  --weight 30"}>
  <CodeBlockHeader>
    <CodeBlockFilename>Create criteria</CodeBlockFilename>
    <CodeBlockActions>
      <CodeBlockCopyButton />
    </CodeBlockActions>
  </CodeBlockHeader>
</CodeBlock>

<Callout type="warn">
Weights should add up to 100. The platform uses weights to calculate the final weighted score for each submission.
</Callout>

## Distribute Submissions

After the submission deadline, assign submissions to judges. Auto-assign distributes them evenly:

<CodeBlock language="bash" showLineNumbers={false} code={"oatmeal judging auto-assign <hackathon> --per-judge 5"}>
  <CodeBlockHeader>
    <CodeBlockFilename>Auto-assign</CodeBlockFilename>
    <CodeBlockActions>
      <CodeBlockCopyButton />
    </CodeBlockActions>
  </CodeBlockHeader>
</CodeBlock>

The `--per-judge` flag controls how many submissions each judge receives. The algorithm distributes evenly so every submission gets reviewed by roughly the same number of judges.

</Callout>
