---
title: Managing Judges & Scoring
description: Set up judging criteria, manage judges, and handle scoring assignments from the CLI.
---

This guide covers the complete judging workflow — defining criteria, inviting judges, distributing submissions, and monitoring scoring progress.

## Define Judging Criteria

Criteria define how judges score submissions. Each criterion has a name, maximum score, and weight (percentage of the final score):

```bash title="Create criteria"
hackathon judging criteria create <hackathon> \
  --name "Innovation" \
  --description "Novelty and creativity of the solution" \
  --max-score 10 \
  --weight 30

hackathon judging criteria create <hackathon> \
  --name "Technical Execution" \
  --description "Code quality, architecture, and completeness" \
  --max-score 10 \
  --weight 40

hackathon judging criteria create <hackathon> \
  --name "Presentation" \
  --description "Demo quality and clarity of explanation" \
  --max-score 10 \
  --weight 30
```

<Callout type="warn">
Weights should add up to 100. The platform uses weights to calculate the final weighted score for each submission.
</Callout>

### Manage Criteria

```bash title="Criteria management"
# List all criteria
hackathon judging criteria list <hackathon>

# Update a criterion
hackathon judging criteria update <hackathon> <criteria-id> --weight 35

# Delete a criterion
hackathon judging criteria delete <hackathon> <criteria-id>
```

## Invite Judges

Add judges by email address. They'll receive an invitation to join:

```bash title="Invite judges"
hackathon judging judges add <hackathon> --email alice@example.com
hackathon judging judges add <hackathon> --email bob@example.com
hackathon judging judges add <hackathon> --email carol@example.com
```

If you have a user's Clerk ID, you can add them directly:

```bash title="Add by user ID"
hackathon judging judges add <hackathon> --user-id user_abc123
```

### Track Invitations

```bash title="Manage invitations"
# See pending invitations
hackathon judging invitations list <hackathon>

# Cancel an invitation
hackathon judging invitations cancel <hackathon> <invitation-id>
```

## Distribute Submissions

After the submission deadline, assign submissions to judges. Auto-assign distributes them evenly:

```bash title="Auto-assign"
hackathon judging auto-assign <hackathon> --per-judge 5
```

The `--per-judge` flag controls how many submissions each judge receives. The algorithm distributes evenly so every submission gets reviewed by roughly the same number of judges.

### Manual Assignments

For special cases, create individual assignments:

```bash title="Manual assignments"
# Assign a specific judge to a specific submission
hackathon judging assignments create <hackathon> \
  --judge <judge-participant-id> \
  --submission <submission-id>

# Remove an assignment
hackathon judging assignments delete <hackathon> <assignment-id>
```

## Monitor Scoring Progress

Track how judging is progressing:

```bash title="Track progress"
# See each judge's progress (e.g., "3/5 scored")
hackathon judging judges list <hackathon>

# View all assignments with scores
hackathon judging assignments list <hackathon>
```

## Subjective Judging (Judge Picks)

If your hackathon uses subjective judging (where judges pick favorites rather than scoring numerically), view the tallies:

```bash title="View pick results"
hackathon judging pick-results <hackathon>
```

## Calculate Results

Once all judges have finished scoring:

```bash title="Finalize"
# Calculate final rankings
hackathon results calculate <hackathon>

# Review before publishing
hackathon results get <hackathon>

# Publish results
hackathon results publish <hackathon>
```

## Remove a Judge

If a judge can no longer participate:

```bash title="Remove judge"
hackathon judging judges remove <hackathon> <judge-participant-id>
```

<Callout type="warn">
Removing a judge also removes their assignments. You may need to re-run auto-assign to redistribute their submissions.
</Callout>

## JSON Output for Reporting

Export judging data as JSON for spreadsheets or custom reporting:

```bash title="Export data"
# Export judge progress
hackathon judging judges list <hackathon> --json > judges.json

# Export all assignments with scores
hackathon judging assignments list <hackathon> --json > assignments.json

# Export final results
hackathon results get <hackathon> --json > results.json
```
