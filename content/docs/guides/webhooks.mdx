Set up webhooks to get notified when agent runs start, complete, or fail.

## Overview

Webhooks allow you to:
- Receive real-time notifications for agent events
- Avoid polling for run completion
- Build event-driven integrations

## Creating a Webhook

```typescript
import { createClient } from "@agents-server/sdk"

const client = createClient(process.env.AGENTS_API_KEY!)

const { data: webhook } = await client.webhooks.create({
  url: "https://your-app.com/webhooks/agents",
  events: ["agent_run.completed", "agent_run.failed"]
})

// Save the secret for verifying signatures
console.log(`Webhook secret: ${webhook.secret}`)
```

<Callout type="warn">
The webhook secret is only shown once. Store it securely for signature verification.
</Callout>

## Available Events

| Event | Description |
|-------|-------------|
| `agent_run.started` | Agent run has started |
| `agent_run.completed` | Agent run completed successfully |
| `agent_run.failed` | Agent run failed |
| `agent_run.step_completed` | A step within the run completed |

## Webhook Payload

Webhook requests include:
- `event` - The event type
- `data` - Event-specific data including `runId`, `agentId`, and results

```json
{
  "event": "agent_run.completed",
  "data": {
    "runId": "run_abc123",
    "agentId": "agent_xyz",
    "status": "succeeded",
    "result": { ... }
  }
}
```

## Handling Webhooks

### Express Example

```typescript
import express from "express"
import crypto from "crypto"

const app = express()
app.use(express.json())

const WEBHOOK_SECRET = process.env.WEBHOOK_SECRET!

app.post("/webhooks/agents", (req, res) => {
  // Verify signature
  const signature = req.headers["x-webhook-signature"] as string
  const payload = JSON.stringify(req.body)
  const expected = crypto
    .createHmac("sha256", WEBHOOK_SECRET)
    .update(payload)
    .digest("hex")

  if (signature !== expected) {
    return res.status(401).send("Invalid signature")
  }

  // Handle event
  const { event, data } = req.body

  switch (event) {
    case "agent_run.completed":
      console.log(`Run ${data.runId} completed:`, data.result)
      break
    case "agent_run.failed":
      console.error(`Run ${data.runId} failed:`, data.error)
      break
  }

  res.status(200).send("OK")
})
```

### Next.js API Route

```typescript
import { NextRequest, NextResponse } from "next/server"
import crypto from "crypto"

export async function POST(req: NextRequest) {
  const signature = req.headers.get("x-webhook-signature")
  const body = await req.text()

  const expected = crypto
    .createHmac("sha256", process.env.WEBHOOK_SECRET!)
    .update(body)
    .digest("hex")

  if (signature !== expected) {
    return NextResponse.json({ error: "Invalid signature" }, { status: 401 })
  }

  const { event, data } = JSON.parse(body)

  if (event === "agent_run.completed") {
    // Process completed run
    await processResult(data.runId, data.result)
  }

  return NextResponse.json({ received: true })
}
```

## Managing Webhooks

### List Webhooks

```typescript
const { data } = await client.webhooks.list()

for (const webhook of data.webhooks) {
  console.log(`${webhook.url} - ${webhook.events.join(", ")}`)
}
```

### Delete a Webhook

```typescript
await client.webhooks.delete(webhookId)
```

## Best Practices

1. **Always verify signatures** - Reject requests with invalid signatures
2. **Respond quickly** - Return 200 within 30 seconds to avoid timeouts
3. **Process asynchronously** - Queue heavy processing for background jobs
4. **Handle retries** - Webhooks may be retried on failure; use idempotency keys
