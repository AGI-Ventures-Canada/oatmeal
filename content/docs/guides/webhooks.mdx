Set up webhooks to get notified when hackathons are created or updated, and when submissions are created or submitted.

## Overview

Webhooks allow you to:
- Receive real-time notifications for hackathon and submission events
- Avoid polling for status changes
- Build event-driven integrations

## Available Events

| Event | Description |
|-------|-------------|
| `hackathon.created` | A new hackathon was created |
| `hackathon.updated` | A hackathon's settings were updated |
| `participant.registered` | A user registered for a hackathon |
| `submission.created` | A team submitted a project |
| `submission.created` | A submission was finalized and submitted |
| `submission.updated` | A team updated their submission |
| `results.published` | Hackathon results were published |

## Webhook Payload

Webhook requests include:
- `event` - The event type
- `timestamp` - ISO 8601 timestamp of when the event occurred
- `data` - Event-specific data including resource IDs and metadata

## Verifying Signatures

Webhook requests include an `X-Webhook-Signature` header containing an HMAC-SHA256 hex digest of the request body, signed with your webhook secret.

### Express Example

<CodeBlock language="typescript" showLineNumbers code={"import express from \"express\"\nimport crypto from \"crypto\"\n\nconst app = express()\napp.use(express.json())\n\nconst WEBHOOK_SECRET = process.env.WEBHOOK_SECRET!\n\napp.post(\"/webhooks/oatmeal\", (req, res) => {\n  const signature = req.headers[\"x-webhook-signature\"] as string\n  const payload = JSON.stringify(req.body)\n  const expected = crypto\n    .createHmac(\"sha256\", WEBHOOK_SECRET)\n    .update(payload)\n    .digest(\"hex\")\n\n  if (signature !== expected) {\n    return res.status(401).send(\"Invalid signature\")\n  }\n\n  const { event, data } = req.body\n\n  switch (event) {\n    case \"hackathon.created\":\n      console.log(`Hackathon created: ${data.hackathonId}`)\n      break\n    case \"submission.created\":\n      console.log(`Submission received: ${data.submissionId}`)\n      break\n  }\n\n  res.status(200).send(\"OK\")\n})"}>

## Best Practices

1. **Always verify signatures** - Reject requests with invalid signatures
2. **Respond quickly** - Return 200 within 30 seconds to avoid timeouts
3. **Process asynchronously** - Queue heavy processing for background jobs
4. **Handle retries** - Webhooks may be retried on failure; use idempotency keys
