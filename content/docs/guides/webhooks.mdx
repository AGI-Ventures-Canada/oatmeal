Set up webhooks to get notified when agent runs start, complete, or fail.

## Overview

Webhooks allow you to:
- Receive real-time notifications for agent events
- Avoid polling for run completion
- Build event-driven integrations

## Available Events

| Event | Description |
|-------|-------------|
| `agent_run.started` | Agent run has started |
| `agent_run.completed` | Agent run completed successfully |
| `agent_run.failed` | Agent run failed |
| `agent_run.step_completed` | A step within the run completed |

## Webhook Payload

Webhook requests include:
- `event` - The event type
- `data` - Event-specific data including `runId`, `agentId`, and results

## Handling Webhooks

### Express Example

<CodeBlock language="typescript" showLineNumbers code={"import express from \"express\"\nimport crypto from \"crypto\"\n\nconst app = express()\napp.use(express.json())\n\nconst WEBHOOK_SECRET = process.env.WEBHOOK_SECRET!\n\napp.post(\"/webhooks/agents\", (req, res) => {\n  // Verify signature\n  const signature = req.headers[\"x-webhook-signature\"] as string\n  const payload = JSON.stringify(req.body)\n  const expected = crypto\n    .createHmac(\"sha256\", WEBHOOK_SECRET)\n    .update(payload)\n    .digest(\"hex\")\n\n  if (signature !== expected) {\n    return res.status(401).send(\"Invalid signature\")\n  }\n\n  // Handle event\n  const { event, data } = req.body\n\n  switch (event) {\n    case \"agent_run.completed\":\n      console.log(`Run ${data.runId} completed:`, data.result)\n      break\n    case \"agent_run.failed\":\n      console.error(`Run ${data.runId} failed:`, data.error)\n      break\n  }\n\n  res.status(200).send(\"OK\")\n})"}>

## Best Practices

1. **Always verify signatures** - Reject requests with invalid signatures
2. **Respond quickly** - Return 200 within 30 seconds to avoid timeouts
3. **Process asynchronously** - Queue heavy processing for background jobs
4. **Handle retries** - Webhooks may be retried on failure; use idempotency keys
