Use AI agents to evaluate hackathon submissions consistently and at scale.

## Overview

Hackathon judging agents can:
- Evaluate projects against defined criteria
- Provide detailed feedback to participants
- Score submissions consistently
- Process many submissions in parallel

## Batch Judging

Process multiple submissions in parallel:

<CodeBlock language="typescript" showLineNumbers code={"const submissions = [\n  { projectName: \"EcoTrack\", /* ... */ },\n  { projectName: \"CodeBuddy\", /* ... */ },\n  { projectName: \"HealthPal\", /* ... */ }\n]\n\n// Start all evaluations in parallel\nconst runs = await Promise.all(\n  submissions.map(submission =>\n    client.agents.run(agent.id, {\n      prompt: \"Evaluate this hackathon submission\",\n      context: submission,\n      idempotencyKey: `judge-${submission.projectName}`\n    })\n  )\n)\n\n// Wait for all results\nconst results = await Promise.all(\n  runs.map(({ data }) => client.agents.waitForResult(data.runId))\n)\n\n// Rank by weighted score\nconst ranked = results\n  .map(r => JSON.parse(r.result))\n  .sort((a, b) => b.weightedScore - a.weightedScore)\n\nconsole.log(\"Rankings:\")\nranked.forEach((project, i) => {\n  console.log(`${i + 1}. ${project.projectName}: ${project.weightedScore}`)\n})"}>

## Best Practices

1. **Calibrate first** - Have judges evaluate 2-3 submissions together to align
2. **Use idempotency keys** - Prevent duplicate evaluations
3. **Review outliers** - Manually check scores that deviate significantly
4. **Provide feedback** - Every participant should get constructive comments
5. **Keep records** - Store all evaluations for transparency
