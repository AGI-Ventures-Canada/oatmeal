This guide walks you through signing in, verifying your setup, and running your first commands with the Oatmeal CLI.

## Sign In

The fastest way to authenticate is the browser flow. Run `oatmeal login` and a browser window opens for you to sign in:

<CodeBlock language="bash" showLineNumbers={false} code={"oatmeal login"}>
  <CodeBlockHeader>
    <CodeBlockFilename>Terminal</CodeBlockFilename>
    <CodeBlockActions>
      <CodeBlockCopyButton />
    </CodeBlockActions>
  </CodeBlockHeader>
</CodeBlock>

The CLI automatically creates an API key named "Oatmeal CLI" with full management scopes and saves it to `~/.oatmeal/config.json`.

### Alternative Auth Methods

If you can't open a browser (e.g., SSH session or CI), use one of these alternatives:

<CodeBlock language="bash" showLineNumbers code={"# Paste an existing API key interactively\noatmeal login --no-browser\n\n# Provide an API key directly\noatmeal login --api-key sk_live_your_key_here\n\n# Use an environment variable (no login needed)\nexport OATMEAL_API_KEY=sk_live_your_key_here\noatmeal hackathons list"}>
  <CodeBlockHeader>
    <CodeBlockFilename>Alternative auth</CodeBlockFilename>
    <CodeBlockActions>
      <CodeBlockCopyButton />
    </CodeBlockActions>
  </CodeBlockHeader>
</CodeBlock>

<CodeBlock language="bash" showLineNumbers={false} code={"oatmeal hackathons create"}>

## Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `OATMEAL_API_KEY` | API key (overrides saved config) | — |
| `OATMEAL_BASE_URL` | API base URL | `https://oatmeal.app` |
| `NO_COLOR` | Disable colored output | — |

## Connecting to Different Environments

The base URL is saved after login. To switch environments:

<CodeBlock language="bash" showLineNumbers code={"# Local development\noatmeal login --base-url http://localhost:3000\n\n# Staging\noatmeal login --base-url https://staging.oatmeal.app\n\n# Production (default)\noatmeal login"}>

## Sign Out

Remove saved credentials:

<CodeBlock language="bash" showLineNumbers={false} code={"oatmeal logout"}>
  <CodeBlockHeader>
    <CodeBlockFilename>Terminal</CodeBlockFilename>
    <CodeBlockActions>
      <CodeBlockCopyButton />
    </CodeBlockActions>
  </CodeBlockHeader>
</CodeBlock>

This deletes `~/.oatmeal/config.json`. It does not revoke the API key — you can do that from the dashboard under Settings > API Keys.
