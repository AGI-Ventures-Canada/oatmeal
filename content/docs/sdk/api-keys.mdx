---
title: API Keys
description: Manage API keys for programmatic access to the Oatmeal platform.
---

API keys are managed from the dashboard at **Settings > API Keys**, or via the CLI with `hackathon login`.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/dashboard/keys` | List API keys |
| `POST` | `/api/dashboard/keys` | Create an API key |
| `POST` | `/api/dashboard/keys/:id/revoke` | Revoke an API key |

<Callout type="info">
API key management endpoints require a Clerk session — they cannot be accessed with an API key. This prevents keys from creating or revoking other keys. Use the dashboard or CLI to manage keys.
</Callout>

## Key Format

API keys use the format `sk_live_` followed by a random string. The full key is only shown once at creation — Oatmeal stores an HMAC-SHA256 hash, not the raw key.

## Available Scopes

| Scope | Description |
|-------|-------------|
| `hackathons:read` | Read hackathon data, submissions, results |
| `hackathons:write` | Create, update, delete hackathons and related data |
| `webhooks:read` | List webhooks |
| `webhooks:write` | Create, delete webhooks |
| `keys:read` | List API keys |
| `keys:write` | Create and revoke API keys |
| `org:read` | Read organization profile |
| `org:write` | Update organization profile |

## List API Keys

`GET /api/dashboard/keys` — requires Clerk session with `keys:read` scope.

List all API keys for your organization from the dashboard at **Settings > API Keys**.

### Response

```json title="response.json"
{
  "keys": [
    {
      "id": "key_abc123",
      "name": "CI/CD Pipeline",
      "prefix": "sk_live_abc1",
      "scopes": ["hackathons:read", "hackathons:write"],
      "createdAt": "2026-02-15T10:00:00Z",
      "lastUsedAt": "2026-03-18T08:30:00Z",
      "revokedAt": null
    }
  ]
}
```

## Create an API Key

`POST /api/dashboard/keys` — requires Clerk session with `keys:write` scope.

Create a new API key from the dashboard at **Settings > API Keys**.

<Callout type="warn">
The full API key is returned only once at creation. Store it securely — it cannot be retrieved again.
</Callout>

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `string` | Yes | Descriptive name for the key |
| `scopes` | `string[]` | No | Permission scopes (defaults to all scopes) |

### Response

```json title="response.json"
{
  "id": "key_abc123",
  "name": "CI/CD Pipeline",
  "prefix": "sk_live_abc1",
  "scopes": ["hackathons:read", "hackathons:write", "webhooks:write"],
  "createdAt": "2026-03-18T10:00:00Z",
  "key": "oat_key_abc123def456ghi789jkl012mno345"
}
```

## Revoke an API Key

`POST /api/dashboard/keys/:id/revoke` — requires Clerk session with `keys:write` scope.

Revoke a key from the dashboard at **Settings > API Keys**. Revoked keys immediately stop working. This action cannot be undone.

<Callout type="info">
**CLI:** `hackathon login` authenticates the CLI and creates a session without managing API keys directly.
</Callout>
