---
title: Submissions API
description: Export and browse project submissions via the REST API.
---

Participants submit projects through the web interface. As an organizer, use the public API to export submission data for reporting or CRM sync.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/public/hackathons/:slug/submissions` | Create a submission |
| `PATCH` | `/api/public/hackathons/:slug/submissions` | Update your submission |
| `GET` | `/api/public/hackathons/:slug/submissions` | List all submissions |
| `GET` | `/api/public/hackathons/:slug/submissions/me` | Get your submission |
| `POST` | `/api/public/hackathons/:slug/submissions/screenshot` | Upload screenshot |
| `DELETE` | `/api/public/hackathons/:slug/submissions/screenshot` | Remove screenshot |

<Callout type="info">
Most submission endpoints require a Clerk session because they are participant actions performed through the web UI. The public `GET` endpoint for listing submissions requires no authentication and is the primary way to export data.
</Callout>

## List All Submissions

`GET /api/public/hackathons/:slug/submissions` — no authentication required.

Returns all submissions for a hackathon, including the team or participant name as `submitter`.

```bash title="List all submissions"
curl -s "$BASE_URL/api/public/hackathons/ai-innovation-2026/submissions" | jq .
```

### Response

```json title="response.json"
{
  "submissions": [

    }
```

## Create a Submission

`POST /api/public/hackathons/:slug/submissions` — requires Clerk session.

This is a participant action. Participants create submissions through the hackathon event page after registering. Each participant or team can only submit once.

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | `string` | Yes | Project title (1-100 characters) |
| `description` | `string` | Yes | Short description (1-280 characters) |
| `githubUrl` | `string` | Yes | GitHub repository URL (must be from github.com) |
| `liveAppUrl` | `string` | No | Live demo URL |

### Error Codes

| Status | Code | Description |
|--------|------|-------------|
| 401 | `not_authenticated` | Sign in required |
| 403 | `not_registered` | Must register before submitting |
| 400 | `submissions_closed` | Submissions are not currently open |
| 400 | `invalid_github_url` | GitHub URL must be from github.com |
| 409 | `already_submitted` | You have already submitted a project |

## Update Your Submission

`PATCH /api/public/hackathons/:slug/submissions` — requires Clerk session.

This is a participant action. Participants update their submissions through the web interface. All fields are optional — only the fields included in the request body are changed.

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | `string` | No | Project title (1-100 characters) |
| `description` | `string` | No | Short description (1-280 characters) |
| `githubUrl` | `string` | No | GitHub repository URL |
| `liveAppUrl` | `string \| null` | No | Live demo URL (null to remove) |

## Get Your Submission

`GET /api/public/hackathons/:slug/submissions/me` — optional Clerk session.

This is a participant action. Returns the authenticated user's submission if one exists, or `null` if not authenticated or no submission found. Participants view their own submission status through the event page.

### Response

```json title="response.json"
{
  "submission": {
    "id": "sub_xyz456",
    "title": "AI Code Review Bot",
    "description": "An AI-powered tool that reviews pull requests",
    "githubUrl": "https://github.com/team/ai-review-bot",
    "liveAppUrl": "https://ai-review-bot.vercel.app",
    "screenshotUrl": null,
    "status": "submitted",
    "createdAt": "2026-04-02T14:30:00Z",
    "updatedAt": "2026-04-02T16:00:00Z"
  }
}
```

## Upload Screenshot

`POST /api/public/hackathons/:slug/submissions/screenshot` — requires Clerk session.

This is a participant action. Participants upload a screenshot through the submission form on the event page. Accepts PNG, JPEG, or WebP images up to 10MB.

### Response

```json title="response.json"
{
  "success": true,
  "screenshotUrl": "https://storage.example.com/screenshots/sub_xyz456.png"
}
```

## Remove Screenshot

`DELETE /api/public/hackathons/:slug/submissions/screenshot` — requires Clerk session.

This is a participant action. Participants remove a screenshot through the submission form on the event page.

## Full Example

Export all submissions for a hackathon to a JSON file:

```bash title="Export submissions"
curl -s "$BASE_URL/api/public/hackathons/ai-innovation-2026/submissions" | jq '.submissions' > submissions.json
```

<Callout type="info">
**CLI:** `hackathon browse submissions ai-innovation-2026 --json`
</Callout>
