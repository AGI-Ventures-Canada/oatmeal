---
title: Schedules API
description: Create and manage recurring jobs via the REST API.
---

The Schedules API lets you create recurring jobs that execute on a defined frequency. Schedules are managed through the dashboard API namespace.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/dashboard/schedules` | List schedules |
| `POST` | `/api/dashboard/schedules` | Create a schedule |
| `GET` | `/api/dashboard/schedules/:id` | Get schedule details |
| `PATCH` | `/api/dashboard/schedules/:id` | Update a schedule |
| `DELETE` | `/api/dashboard/schedules/:id` | Delete a schedule |

All endpoints require an API key with `schedules:read` or `schedules:write` scope.

## Create a Schedule

`POST /api/dashboard/schedules` — requires `schedules:write` scope.

```bash title="Create a schedule"
curl -s -X POST "$BASE_URL/api/dashboard/schedules" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Daily report",
    "frequency": "daily",
    "timezone": "America/New_York",
    "jobType": "generate-report",
    "input": {"format": "pdf"}
  }' | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon schedules create --name "Daily report" --frequency daily --timezone America/New_York
```
</Callout>

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `string` | Yes | Schedule name |
| `frequency` | `string` | Yes | One of: `once`, `hourly`, `daily`, `weekly`, `monthly`, `cron` |
| `cronExpression` | `string` | No | Cron expression (required when frequency is `cron`) |
| `timezone` | `string` | No | IANA timezone (e.g. `America/New_York`) |
| `runTime` | `string` | No | Time of day to run (e.g. `09:00`) |
| `jobType` | `string` | No | Job type to execute on each run |
| `input` | `object` | No | Input payload passed to each job |

### Frequency Options

| Frequency | Description |
|-----------|-------------|
| `once` | Run a single time |
| `hourly` | Run every hour |
| `daily` | Run once per day |
| `weekly` | Run once per week |
| `monthly` | Run once per month |
| `cron` | Custom schedule using a cron expression |

### Response

```json title="response.json"
{
  "id": "sched_abc123",
  "name": "Daily report",
  "nextRunAt": "2026-02-11T14:00:00Z"
}
```

## List Schedules

`GET /api/dashboard/schedules` — requires `schedules:read` scope.

Supports optional query parameters: `limit` (number) and `activeOnly` (`true`/`false`).

```bash title="List active schedules"
curl -s "$BASE_URL/api/dashboard/schedules?activeOnly=true" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon schedules list
```
</Callout>

### Response

```json title="response.json"
{
  "schedules": [
    {
      "id": "sched_abc123",
      "name": "Daily report",
      "frequency": "daily",
      "cronExpression": null,
      "timezone": "America/New_York",

      "isActive": true,
```

## Update a Schedule

`PATCH /api/dashboard/schedules/:id` — requires `schedules:write` scope.

All fields are optional. Only provided fields are updated.

```bash title="Pause a schedule"
curl -s -X PATCH "$BASE_URL/api/dashboard/schedules/$SCHEDULE_ID" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"isActive": false}' | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon schedules update $SCHEDULE_ID --active false
```
</Callout>

### Updatable Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | `string` | Schedule name |
| `frequency` | `string` | Frequency preset or `cron` |
| `cronExpression` | `string` | Cron expression |
| `timezone` | `string` | IANA timezone |
| `runTime` | `string` | Time of day |
| `input` | `object` | Job input payload |
| `isActive` | `boolean` | Enable or disable the schedule |

### Response

```json title="response.json"
{
  "id": "sched_abc123",
  "nextRunAt": null,
  "updatedAt": "2026-02-10T16:00:00Z"
}
```

## Delete a Schedule

`DELETE /api/dashboard/schedules/:id` — requires `schedules:write` scope.

```bash title="Delete a schedule"
curl -s -X DELETE "$BASE_URL/api/dashboard/schedules/$SCHEDULE_ID" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon schedules delete $SCHEDULE_ID
```
</Callout>

## Cron Expression Examples

When using `frequency: "cron"`, provide a standard cron expression:

| Expression | Description |
|------------|-------------|
| `0 9 * * 1-5` | Weekdays at 9:00 AM |
| `0 */2 * * *` | Every 2 hours |
| `0 0 1 * *` | First day of each month at midnight |
| `30 14 * * 3` | Wednesdays at 2:30 PM |
| `0 8 * * 1` | Mondays at 8:00 AM |
