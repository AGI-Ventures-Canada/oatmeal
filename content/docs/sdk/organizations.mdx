---
title: Organizations API
description: Manage organization profiles, logos, and public pages via the REST API.
---

The Organizations API lets you manage your organization profile, upload logos, and check slug availability. Organizations are the top-level tenant in Oatmeal — all hackathons, API keys, and integrations belong to an organization.

## Endpoints

### Dashboard (API key or session)

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/dashboard/org-profile` | Get organization profile |
| `PATCH` | `/api/dashboard/org-profile` | Update organization profile |
| `POST` | `/api/dashboard/upload-logo` | Upload logo |
| `DELETE` | `/api/dashboard/logo/:variant` | Delete logo |
| `GET` | `/api/dashboard/organizations/slug-available` | Check slug availability |

### Public (no auth required)

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/public/orgs/:slug` | Get public organization page |

Dashboard endpoints require an API key with `org:read` or `org:write` scope, or a Clerk session. Slug availability check requires a Clerk session.

## Get Organization Profile

`GET /api/dashboard/org-profile` — requires `org:read` scope.

```bash title="Get organization profile"
curl -s "$BASE_URL/api/dashboard/org-profile" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

### Response

```json title="response.json"
{
  "id": "org_abc123",
  "name": "AGI Ventures",
  "slug": "agi-ventures",
  "logoUrl": "https://storage.example.com/logos/org_abc123_light.png",
  "description": "Building the future of hackathons",
  "websiteUrl": "https://agiventures.com"
}
```

## Update Organization Profile

`PATCH /api/dashboard/org-profile` — requires `org:write` scope.

```bash title="Update organization profile"
curl -s -X PATCH "$BASE_URL/api/dashboard/org-profile" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "AGI Ventures Canada",
    "description": "Building the future of hackathons in Canada",
    "websiteUrl": "https://agiventures.ca"
  }' | jq .
```

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `string` | No | Organization name |
| `slug` | `string` | No | URL slug (min 3 chars, lowercase letters/numbers/hyphens) |
| `description` | `string \| null` | No | Organization description |
| `websiteUrl` | `string \| null` | No | Website URL |
| `logoUrl` | `string \| null` | No | Light mode logo URL |
| `logoUrlDark` | `string \| null` | No | Dark mode logo URL |

## Upload Logo

`POST /api/dashboard/upload-logo` — requires `org:write` scope.

Accepts PNG, JPEG, WebP, or SVG files up to 30MB. Specify the variant to upload separate light and dark mode logos.

```bash title="Upload a light-mode logo"
curl -s -X POST "$BASE_URL/api/dashboard/upload-logo" \
  -H "Authorization: Bearer $API_KEY" \
  -F "file=@logo.png" \
  -F "variant=light" | jq .
```

### Request Body (FormData)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `file` | `File` | Yes | Image file (PNG, JPEG, WebP, or SVG, max 30MB) |
| `variant` | `"light" \| "dark"` | Yes | Logo variant |

## Delete Logo

`DELETE /api/dashboard/logo/:variant` — requires `org:write` scope.

```bash title="Delete the dark-mode logo"
curl -s -X DELETE "$BASE_URL/api/dashboard/logo/dark" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

## Check Slug Availability

`GET /api/dashboard/organizations/slug-available` — requires Clerk session (not API key).

This endpoint is only available from the dashboard UI using a Clerk session cookie. Check slug availability from the dashboard UI when creating or renaming an organization.

## Get Public Organization Page

`GET /api/public/orgs/:slug` — no authentication required.

Returns the organization profile along with all hackathons they've organized or sponsored.

```bash title="Get public organization page"
curl -s "$BASE_URL/api/public/orgs/agi-ventures" | jq .
```
