---
title: Registration & Teams API
description: Check registration status and browse team invitations via the REST API.
---

Registration, team formation, and invitations are participant actions handled through the web interface. As an organizer, use the public API to check registration counts for reporting.

## Endpoints

### Public (Clerk session)

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/public/hackathons/:slug/register` | Register for a hackathon |
| `GET` | `/api/public/hackathons/:slug/registration` | Get registration status |
| `GET` | `/api/public/invitations/:token` | Get invitation details |
| `POST` | `/api/public/invitations/:token/accept` | Accept a team invitation |
| `POST` | `/api/public/invitations/:token/decline` | Decline a team invitation |

### Dashboard (Clerk session only)

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/dashboard/teams/:teamId/invitations` | Send a team invitation |
| `GET` | `/api/dashboard/teams/:teamId/invitations` | List team invitations |
| `DELETE` | `/api/dashboard/teams/:teamId/invitations/:id` | Cancel an invitation |

<Callout type="info">
All registration and team invitation endpoints require a Clerk session (not API keys) since they are participant-facing actions performed through the web UI.
</Callout>

## Check Registration Status

`GET /api/public/hackathons/:slug/registration` — optional Clerk session.

Returns the participant count and whether the current user is registered. Without authentication, `isRegistered` is `null`.

```bash title="Check registration status"
curl -s "$BASE_URL/api/public/hackathons/ai-innovation-2026/registration" | jq .
```

### Response

```json title="response.json"
{
  "participantCount": 87,
  "isRegistered": true
}
```

When not authenticated, `isRegistered` is `null`.

## Get Invitation Details

`GET /api/public/invitations/:token` — no authentication required.

Retrieve invitation details by token. Used to show the invitation page before the invitee accepts or declines.

```bash title="Get invitation details"
curl -s "$BASE_URL/api/public/invitations/inv_token_here" | jq .
```

### Response

```json title="response.json"
{
  "id": "inv_abc123",
  "status": "pending",
  "teamName": "Team Alpha",
  "hackathonName": "AI Innovation Challenge 2026",
  "hackathonSlug": "ai-innovation-challenge-2026",
  "hackathonStatus": "registration_open",
  "email": "teammate@example.com",
  "expiresAt": "2026-03-25T00:00:00Z"
}
```

## Register for a Hackathon

`POST /api/public/hackathons/:slug/register` — requires Clerk session.

This is a participant action. Participants register through the hackathon event page. A team is automatically created with the user as captain.

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `latitude` | `number` | No | Location latitude (for in-person verification) |
| `longitude` | `number` | No | Location longitude (for in-person verification) |

### Response

```json title="response.json"
{
  "success": true,
  "participantId": "part_abc123",
  "teamId": "team_xyz456"
}
```

### Error Codes

| Status | Code | Description |
|--------|------|-------------|
| 401 | `not_authenticated` | Sign in required |
| 404 | `hackathon_not_found` | Hackathon does not exist |
| 409 | `already_registered` | User is already registered |
| 400 | — | Registration is not open or location verification failed |

## Send a Team Invitation

`POST /api/dashboard/teams/:teamId/invitations` — requires Clerk session.

This is a participant action. Team captains send invitations through the team management page. Rate limited to 10 invitations per minute per team.

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `hackathonId` | `string` | Yes | Hackathon ID |
| `email` | `string` | Yes | Invitee's email address |
| `inviterName` | `string` | No | Display name of the person inviting |

### Response

```json title="response.json"
{
  "id": "inv_abc123",
  "email": "teammate@example.com",
  "expiresAt": "2026-03-25T00:00:00Z",
  "emailSent": true
}
```

## List Team Invitations

`GET /api/dashboard/teams/:teamId/invitations` — requires Clerk session.

This is a participant action. Team captains view pending and past invitations through the team management page.

### Query Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `status` | `string` | No | Filter by status: `pending`, `accepted`, `declined`, `expired`, `cancelled` |

### Response

```json title="response.json"
{
  "invitations": [
    {
      "id": "inv_abc123",
      "email": "teammate@example.com",
      "status": "pending",
      "expiresAt": "2026-03-25T00:00:00Z",
      "createdAt": "2026-03-18T10:00:00Z"
    }
  ]
}
```

## Cancel an Invitation

`DELETE /api/dashboard/teams/:teamId/invitations/:invitationId` — requires Clerk session.

This is a participant action. Team captains cancel pending invitations through the team management page.

## Accept an Invitation

`POST /api/public/invitations/:token/accept` — requires Clerk session.

This is a participant action. Invitees accept team invitations through the invitation link sent via email. The authenticated user's email must match the invitation email.

### Response

```json title="response.json"
{
  "success": true,
  "teamId": "team_xyz456",
  "hackathonSlug": "ai-innovation-challenge-2026"
}
```

## Decline an Invitation

`POST /api/public/invitations/:token/decline` — requires Clerk session.

This is a participant action. Invitees decline team invitations through the invitation link sent via email.

Check the registration count for a hackathon:
