---
title: Results API
description: Calculate, publish, and retrieve hackathon results and leaderboards via the REST API.
---

The Results API handles scoring aggregation, leaderboard generation, and publication of hackathon results.

## Endpoints

### Dashboard (API key or session)

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/dashboard/hackathons/:id/results/calculate` | Calculate results |
| `GET` | `/api/dashboard/hackathons/:id/results` | Get results (organizer view) |
| `POST` | `/api/dashboard/hackathons/:id/results/publish` | Publish results |
| `POST` | `/api/dashboard/hackathons/:id/results/unpublish` | Unpublish results |

### Public (no auth required)

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/public/hackathons/:slug/results` | Get published results |

Dashboard endpoints require an API key with `hackathons:read` or `hackathons:write` scope.

## Response Types

```typescript title="types.ts"
interface Result {
  id: string
  rank: number
  submissionId: string
  submissionTitle: string
  submissionDescription: string | null
  submissionGithubUrl: string | null
  submissionLiveAppUrl: string | null
  submissionScreenshotUrl: string | null
  submissionTeamId: string | null
  totalScore: number
  weightedScore: number | null
  judgeCount: number
  publishedAt: string | null
  teamName: string | null
  prizes: { id: string; name: string; value: string | null }[]
}
```

## Calculate Results

`POST /api/dashboard/hackathons/:id/results/calculate` — requires `hackathons:write` scope.

Aggregates judge scores (points mode) or judge picks (subjective mode) and generates rankings. Run this after judging is complete.

```bash title="Calculate results"
curl -s -X POST "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/results/calculate" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon results calculate --hackathon "$HACKATHON_ID"
```
</Callout>

### Response

```json title="response.json"
{
  "success": true,
  "count": 12
}
```

<Callout type="info">
You can recalculate results at any time. New calculations replace previous results. This is useful if you add or remove judges, or if scoring needs corrections.
</Callout>

## Get Results (Organizer)

`GET /api/dashboard/hackathons/:id/results` — requires `hackathons:read` scope.

Returns the full leaderboard with scores, regardless of publication status.

```bash title="Get results as organizer"
curl -s "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/results" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon results get --hackathon "$HACKATHON_ID" --json
```
</Callout>

## Publish Results

`POST /api/dashboard/hackathons/:id/results/publish` — requires `hackathons:write` scope.

Makes results visible on the public hackathon page. This also:
- Auto-assigns score-based prizes to top-ranked submissions
- Sends winner notification emails
- Triggers `results.published` webhooks

```bash title="Publish results"
curl -s -X POST "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/results/publish" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon results publish --hackathon "$HACKATHON_ID"
```
</Callout>

<Callout type="warn">
Publishing triggers winner emails and webhooks. Make sure results are finalized before publishing.
</Callout>

## Unpublish Results

`POST /api/dashboard/hackathons/:id/results/unpublish` — requires `hackathons:write` scope.

Hides results from the public page and reverts the hackathon status to `"judging"`.

```bash title="Unpublish results"
curl -s -X POST "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/results/unpublish" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon results unpublish --hackathon "$HACKATHON_ID"
```
</Callout>

## Get Published Results (Public)

`GET /api/public/hackathons/:slug/results` — no authentication required.

Returns the public leaderboard. Returns `404` if results are not yet published.

```bash title="Browse published results"
curl -s "$BASE_URL/api/public/hackathons/ai-innovation-2026/results" | jq .
```

## Full Example

```bash title="Calculate, preview, and publish results"
#!/usr/bin/env bash
set -euo pipefail

# Calculate results
echo "Calculating results..."
CALC=$(curl -s -X POST "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/results/calculate" \
  -H "Authorization: Bearer $API_KEY")
COUNT=$(echo "$CALC" | jq -r '.count')
echo "Ranked $COUNT submissions"

# Preview the leaderboard
echo -e "\nLeaderboard preview:"
curl -s "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/results" \
  -H "Authorization: Bearer $API_KEY" \
  | jq -r '.results[:5][] | "  #\(.rank) \(.submissionTitle) (\(.totalScore) pts)"'

# Publish
echo -e "\nPublishing results..."
curl -s -X POST "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/results/publish" \
  -H "Authorization: Bearer $API_KEY" | jq .

echo "Results published!"
```
