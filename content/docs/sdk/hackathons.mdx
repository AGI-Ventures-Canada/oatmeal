---
title: Hackathons API
description: Create, manage, and discover hackathon events via the REST API.
---

The Hackathons API lets you create events, update settings, manage banners, and transition hackathons through their lifecycle.

## Endpoints

### Dashboard (API key or session)

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/dashboard/hackathons` | Create a hackathon |
| `GET` | `/api/dashboard/hackathons` | List your hackathons |
| `GET` | `/api/dashboard/hackathons/:id` | Get hackathon details |
| `PATCH` | `/api/dashboard/hackathons/:id/settings` | Update hackathon settings |
| `DELETE` | `/api/dashboard/hackathons/:id` | Delete a hackathon |
| `POST` | `/api/dashboard/hackathons/:id/banner` | Upload banner image |
| `DELETE` | `/api/dashboard/hackathons/:id/banner` | Remove banner image |

### Public (no auth required)

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/public/hackathons` | List public hackathons |
| `GET` | `/api/public/hackathons/:slug` | Get hackathon by slug |

Dashboard endpoints require an API key with `hackathons:read` or `hackathons:write` scope. Public endpoints are unauthenticated.

## Status Lifecycle

Hackathons progress through these statuses:

```typescript title="types.ts"
type HackathonStatus =
  | "draft"              // Initial state, not visible to public
  | "published"          // Visible but registration not yet open
  | "registration_open"  // Accepting registrations
  | "active"             // Hackathon is running
  | "judging"            // Judging phase
  | "completed"          // Judging finished
  | "archived"           // Historic record
```

Transition between statuses by updating the `status` field via the settings endpoint.

## Response Types

```typescript title="types.ts"
interface Hackathon {
  id: string
  name: string
  slug: string
  description: string | null
  rules: string | null
  bannerUrl: string | null
  status: HackathonStatus
  startsAt: string | null
  endsAt: string | null
  registrationOpensAt: string | null
  registrationClosesAt: string | null
  maxParticipants: number | null
  minTeamSize: number
  maxTeamSize: number
  allowSolo: boolean
  anonymousJudging: boolean
  judgingMode: "points" | "subjective"
  resultsPublishedAt: string | null
  createdAt: string
  updatedAt: string
}
```

## Create a Hackathon

`POST /api/dashboard/hackathons` — requires `hackathons:write` scope.

```bash title="Create a hackathon"
curl -s -X POST "$BASE_URL/api/dashboard/hackathons" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "AI Innovation Challenge 2026",
    "description": "Build the future of AI-powered tools"
  }' | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon events create --name "AI Innovation Challenge 2026" --description "Build the future of AI-powered tools"
```
</Callout>

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `string` | Yes | Hackathon name (min 1 character) |
| `description` | `string` | No | Short description |

### Response

```json title="response.json"
{
  "id": "hack_abc123",
  "name": "AI Innovation Challenge 2026",
  "slug": "ai-innovation-challenge-2026"
}
```

## Get Hackathon Details

`GET /api/dashboard/hackathons/:id` — requires `hackathons:read` scope.

```bash title="Get hackathon details"
curl -s "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon events get --id "$HACKATHON_ID" --json
```
</Callout>

### Response

Returns the full `Hackathon` object as defined in the Response Types section.

## Update Hackathon Settings

`PATCH /api/dashboard/hackathons/:id/settings` — requires `hackathons:write` scope.

All fields are optional. Only include the fields you want to change.

```bash title="Update hackathon settings"
curl -s -X PATCH "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/settings" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "registration_open",
    "startsAt": "2026-04-01T09:00:00Z",
    "endsAt": "2026-04-03T17:00:00Z",
    "registrationOpensAt": "2026-03-01T00:00:00Z",
    "registrationClosesAt": "2026-03-28T23:59:59Z",
    "maxParticipants": 200,
    "minTeamSize": 2,
    "maxTeamSize": 5,
    "allowSolo": false,
    "judgingMode": "points",
    "anonymousJudging": true
  }' | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon events update --id "$HACKATHON_ID" --status registration_open --max-participants 200
```
</Callout>

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | `string` | No | Hackathon name |
| `description` | `string \| null` | No | Description |
| `rules` | `string \| null` | No | Rules and guidelines |
| `status` | `HackathonStatus` | No | Lifecycle status |
| `startsAt` | `string \| null` | No | Event start (ISO 8601) |
| `endsAt` | `string \| null` | No | Event end (ISO 8601) |
| `registrationOpensAt` | `string \| null` | No | Registration opens (ISO 8601) |
| `registrationClosesAt` | `string \| null` | No | Registration closes (ISO 8601) |
| `maxParticipants` | `number \| null` | No | Max participants (null = unlimited) |
| `minTeamSize` | `number` | No | Minimum team size |
| `maxTeamSize` | `number` | No | Maximum team size |
| `allowSolo` | `boolean` | No | Allow solo participants |
| `judgingMode` | `"points" \| "subjective"` | No | Scoring mode |
| `anonymousJudging` | `boolean` | No | Hide team names from judges |
| `locationType` | `"in_person" \| "virtual" \| null` | No | Location type |
| `locationName` | `string \| null` | No | Venue name |
| `locationUrl` | `string \| null` | No | Virtual location URL |

### Response

Returns the full updated `Hackathon` object.

## Delete a Hackathon

`DELETE /api/dashboard/hackathons/:id` — requires `hackathons:write` scope.

<Callout type="warn">
Deleting a hackathon is permanent and cascades to all participants, submissions, judging data, and results.
</Callout>

```bash title="Delete a hackathon"
curl -s -X DELETE "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon events delete --id "$HACKATHON_ID"
```
</Callout>

## Upload Banner

`POST /api/dashboard/hackathons/:id/banner` — requires `hackathons:write` scope.

Accepts PNG, JPEG, or WebP images up to 50MB.

```bash title="Upload banner image"
curl -s -X POST "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/banner" \
  -H "Authorization: Bearer $API_KEY" \
  -F "file=@banner.webp" | jq .
```

## Remove Banner

`DELETE /api/dashboard/hackathons/:id/banner` — requires `hackathons:write` scope.

```bash title="Remove banner image"
curl -s -X DELETE "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/banner" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

## Full Example

```bash title="Create and configure a hackathon"
#!/usr/bin/env bash
set -euo pipefail

# Create the hackathon
HACKATHON_ID=$(curl -s -X POST "$BASE_URL/api/dashboard/hackathons" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Spring Hack 2026",
    "description": "48-hour sprint to build something amazing"
  }' | jq -r '.id')

echo "Created hackathon: $HACKATHON_ID"

# Configure settings
curl -s -X PATCH "$BASE_URL/api/dashboard/hackathons/$HACKATHON_ID/settings" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "published",
    "startsAt": "2026-05-01T09:00:00Z",
    "endsAt": "2026-05-03T09:00:00Z",
    "registrationOpensAt": "2026-04-01T00:00:00Z",
    "registrationClosesAt": "2026-04-28T23:59:59Z",
    "maxParticipants": 100,
    "minTeamSize": 2,
    "maxTeamSize": 4,
    "allowSolo": true,
    "judgingMode": "points",
    "anonymousJudging": true
  }' | jq .

echo "Hackathon $HACKATHON_ID created and configured"
```
