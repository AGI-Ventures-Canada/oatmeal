The Jobs API allows you to queue async tasks and poll for their results.

<CodeBlock language="typescript" showLineNumbers code={"type JobStatus = \"queued\" | \"running\" | \"succeeded\" | \"failed\" | \"canceled\"\n\ninterface Job {\n  id: string\n  type: string\n  status: JobStatus\n  createdAt: string\n  updatedAt?: string\n  completedAt?: string | null\n}\n\ninterface JobResult {\n  id: string\n  status: JobStatus\n  result: unknown\n  error: unknown\n  completedAt: string\n}\n\ninterface CreateJobInput {\n  type: string\n  input?: unknown\n  idempotencyKey?: string\n}"}>

## Wait for Result

Poll until the job completes (recommended for most use cases):

<CodeBlock language="typescript" showLineNumbers code={"try {\n  const result = await client.jobs.waitForResult(\"job_abc123\", {\n    maxAttempts: 60,  // Default: 60\n    intervalMs: 1000  // Default: 1000ms\n  })\n\n  if (result.status === \"succeeded\") {\n    console.log(\"Success:\", result.result)\n  } else {\n    console.error(\"Failed:\", result.error)\n  }\n} catch (err) {\n  console.error(\"Timeout or error:\", err)\n}"}>

## Full Example
