---
title: Jobs API
description: Create and manage async jobs via the REST API.
---

The Jobs API allows you to queue async tasks and poll for their results.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/v1/jobs` | Create a new job |
| `GET` | `/api/v1/jobs/:id` | Get job status |
| `GET` | `/api/v1/jobs/:id/result` | Get job result |
| `POST` | `/api/v1/jobs/:id/cancel` | Cancel a job |

All endpoints require an API key with `hackathons:read` or `hackathons:write` scope.

## Response Types

```typescript title="types.ts"
type JobStatus = "queued" | "running" | "succeeded" | "failed" | "canceled"

interface Job {
  id: string
  type: string
  status: JobStatus
  createdAt: string
  updatedAt?: string
  completedAt?: string | null
}

interface JobResult {
  id: string
  status: JobStatus
  result: unknown
  error: unknown
  completedAt: string
}
```

## Create a Job

`POST /api/v1/jobs` — requires `hackathons:write` scope.

```typescript title="create-job.ts"
const res = await fetch(`${BASE_URL}/api/v1/jobs`, {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    type: "process-document",
    input: {
      url: "https://example.com/document.pdf",
      options: { extractTables: true },
    },
    idempotencyKey: "doc-abc-123",
  }),
})

const job = await res.json()
console.log("Job created:", job.id)
console.log("Status:", job.status) // "queued"
```

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | `string` | Yes | Job type identifier |
| `input` | `object` | No | Job input payload (JSON) |
| `idempotencyKey` | `string` | No | Unique key to prevent duplicate jobs |

The idempotency key can also be sent as an `Idempotency-Key` HTTP header.

### Response

```json title="response.json"
{
  "id": "job_abc123",
  "type": "process-document",
  "status": "queued",
  "createdAt": "2026-02-10T15:30:00Z"
}
```

## Get Job Status

`GET /api/v1/jobs/:id` — requires `hackathons:read` scope.

```typescript title="get-status.ts"
const res = await fetch(`${BASE_URL}/api/v1/jobs/${jobId}`, {
  headers: {
    "Authorization": `Bearer ${API_KEY}`,
  },
})

const job = await res.json()
console.log("Status:", job.status)
console.log("Created:", job.createdAt)
```

### Response

```json title="response.json"
{
  "id": "job_abc123",
  "type": "process-document",
  "status": "running",
  "createdAt": "2026-02-10T15:30:00Z",
  "updatedAt": "2026-02-10T15:30:05Z",
  "completedAt": null
}
```

## Get Job Result

`GET /api/v1/jobs/:id/result` — requires `hackathons:read` scope.

Returns `202` with `{ "error": "Job not completed", "status": "running" }` if the job is still in progress.

```typescript title="get-result.ts"
const res = await fetch(`${BASE_URL}/api/v1/jobs/${jobId}/result`, {
  headers: {
    "Authorization": `Bearer ${API_KEY}`,
  },
})

if (res.status === 202) {
  console.log("Job still processing...")
} else {
  const data = await res.json()
  console.log("Result:", data.result)
}
```

### Response (completed)

```json title="response.json"
{
  "id": "job_abc123",
  "status": "succeeded",
  "result": { "pages": 12, "tables": 3 },
  "error": null,
  "completedAt": "2026-02-10T15:31:00Z"
}
```

## Polling Helper

For convenience, here is a simple polling function you can use to wait for job completion:

```typescript title="polling-helper.ts"
async function waitForResult(
  jobId: string,
  options?: { maxAttempts?: number; intervalMs?: number }
) {
  const { maxAttempts = 60, intervalMs = 1000 } = options ?? {}

  for (let i = 0; i < maxAttempts; i++) {
    const res = await fetch(`${BASE_URL}/api/v1/jobs/${jobId}/result`, {
      headers: { "Authorization": `Bearer ${API_KEY}` },
    })

    if (res.status !== 202) {
      return await res.json()
    }

    await new Promise((r) => setTimeout(r, intervalMs))
  }

  throw new Error(`Job ${jobId} did not complete within ${maxAttempts} attempts`)
}
```

## Cancel a Job

`POST /api/v1/jobs/:id/cancel` — requires `hackathons:write` scope.

Cancels a queued or running job. Cannot cancel already-completed jobs.

```typescript title="cancel-job.ts"
const res = await fetch(`${BASE_URL}/api/v1/jobs/${jobId}/cancel`, {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${API_KEY}`,
  },
})

const data = await res.json()
if (data.success) {
  console.log("Job canceled")
}
```

## Full Example

```typescript title="full-example.ts"
const API_KEY = process.env.OATMEAL_API_KEY!
const BASE_URL = process.env.OATMEAL_BASE_URL!

async function processDocument(url: string) {
  const res = await fetch(`${BASE_URL}/api/v1/jobs`, {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      type: "process-document",
      input: { url },
    }),
  })

  if (!res.ok) {
    const err = await res.json()
    throw new Error(err.error)
  }

  const job = await res.json()

  const result = await waitForResult(job.id)

  if (result.status === "failed") {
    throw new Error(String(result.error))
  }

  return result.result
}
```
