The Webhooks API allows you to register URLs that receive real-time event notifications.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/v1/webhooks` | List all webhooks |
| `POST` | `/api/v1/webhooks` | Create a webhook |
| `DELETE` | `/api/v1/webhooks/:id` | Delete a webhook |

All endpoints require an API key with `webhooks:read` or `webhooks:write` scope.

## Available Events

| Event | Description |
|-------|-------------|
| `hackathon.created` | A new hackathon was created |
| `hackathon.updated` | A hackathon's settings were updated |
| `participant.registered` | A user registered for a hackathon |
| `submission.created` | A team created a new submission |
| `submission.submitted` | A submission was finalized and submitted |
| `submission.updated` | A team updated their submission |
| `results.published` | Hackathon results were published |

## Create a Webhook

`POST /api/v1/webhooks` — requires `webhooks:write` scope.

```bash title="Create a webhook"
curl -s -X POST "$BASE_URL/api/v1/webhooks" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-app.com/webhooks/oatmeal",
    "events": ["hackathon.created", "submission.created"]
  }' | jq .
```

<Callout type="warn">
The webhook `secret` is only returned once at creation time. Store it securely for signature verification.
</Callout>

<Callout type="info" title="CLI equivalent">
```bash
hackathon webhooks create
```
</Callout>

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `url` | `string` | Yes | Webhook endpoint URL (must be a valid URI) |
| `events` | `string[]` | Yes | Events to subscribe to |

### Response

```json title="response.json"
{
  "id": "wh_abc123",
  "url": "https://your-app.com/webhooks/oatmeal",
  "events": ["hackathon.created", "submission.created"],
  "secret": "whsec_...",
  "createdAt": "2026-02-10T15:30:00Z"
}
```

## List Webhooks

`GET /api/v1/webhooks` — requires `webhooks:read` scope.

```bash title="List webhooks"
curl -s "$BASE_URL/api/v1/webhooks" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon webhooks list
```
</Callout>

### Response

```json title="response.json"
{
  "webhooks": [
    {
      "id": "wh_abc123",
      "url": "https://your-app.com/webhooks/oatmeal",
      "events": ["hackathon.created", "submission.created"],
      "isActive": true,
      "failureCount": 0,
      "lastTriggeredAt": "2026-02-10T16:00:00Z",
      "createdAt": "2026-02-10T15:30:00Z"
    }
  ]
}
```

## Delete a Webhook

`DELETE /api/v1/webhooks/:id` — requires `webhooks:write` scope.

```bash title="Delete a webhook"
curl -s -X DELETE "$BASE_URL/api/v1/webhooks/$WEBHOOK_ID" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon webhooks delete
```
</Callout>

## Webhook Payload Format

Webhook requests are sent as `POST` with a JSON body:

```json title="payload.json"
{
  "event": "submission.created",
  "timestamp": "2026-02-10T15:30:00Z",
  "data": {
    "hackathonId": "hack_abc123",
    "submissionId": "sub_xyz456",
    "title": "AI Code Review Bot"
  }
}
```

Requests include an `X-Webhook-Signature` header containing an HMAC-SHA256 hex digest of the request body, signed with your webhook secret. See the [Webhooks Guide](/docs/guides/webhooks) for verification examples.
