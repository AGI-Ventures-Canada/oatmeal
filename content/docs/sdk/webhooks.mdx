The Webhooks API allows you to register URLs that receive real-time event notifications.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/v1/webhooks` | List all webhooks |
| `POST` | `/api/v1/webhooks` | Create a webhook |
| `DELETE` | `/api/v1/webhooks/:id` | Delete a webhook |

All endpoints require an API key with `webhooks:read` or `webhooks:write` scope.

## Available Events

| Event | Description |
|-------|-------------|
| `hackathon.created` | A new hackathon was created |
| `hackathon.updated` | A hackathon's settings were updated |
| `submission.created` | A new submission draft was created |
| `submission.submitted` | A submission was finalized and submitted |

## Create a Webhook

`POST /api/v1/webhooks` — requires `webhooks:write` scope.

<CodeBlock language="typescript" showLineNumbers code={"const res = await fetch(`${BASE_URL}/api/v1/webhooks`, {\n  method: \"POST\",\n  headers: {\n    \"Authorization\": `Bearer ${API_KEY}`,\n    \"Content-Type\": \"application/json\",\n  },\n  body: JSON.stringify({\n    url: \"https://your-app.com/webhooks/oatmeal\",\n    events: [\"hackathon.created\", \"submission.submitted\"],\n  }),\n})\n\nconst webhook = await res.json()\nconsole.log(\"Webhook ID:\", webhook.id)\nconsole.log(\"Secret:\", webhook.secret)"}>

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `url` | `string` | Yes | Webhook endpoint URL (must be a valid URI) |
| `events` | `string[]` | Yes | Events to subscribe to |

## Webhook Payload Format

Webhook requests are sent as `POST` with a JSON body:

<CodeBlock language="json" code={"{\n  \"event\": \"submission.submitted\",\n  \"data\": {\n    \"hackathonId\": \"hack_abc123\",\n    \"submissionId\": \"sub_xyz456\",\n    \"title\": \"AI Code Review Bot\",\n    \"teamName\": \"BuilderSquad\",\n    \"submittedAt\": \"2026-02-10T15:30:00Z\"\n  }\n}"}>
  <CodeBlockHeader>

  </CodeBlockHeader>

Requests include an `X-Webhook-Signature` header containing an HMAC-SHA256 hex digest of the request body, signed with your webhook secret. See the [Webhooks Guide](/docs/guides/webhooks) for verification examples.
