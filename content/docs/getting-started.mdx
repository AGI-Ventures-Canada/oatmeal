The Oatmeal platform exposes a REST API that you can call with `curl`, the [CLI](/docs/cli), or any HTTP client. For a comprehensive overview of all authentication methods, see the [Authentication](/docs/authentication) guide.

## Get Your API Key

Create an API key from the dashboard at **Settings > API Keys**. Keys start with `sk_live_` and are shown only once.

<Callout type="warn">
Never expose your API key in client-side code. Use environment variables and server-side API routes.
</Callout>

## Environment Variables

Store your API key and base URL as shell variables:

```bash title="Setup"
export BASE_URL="https://your-app.com"   # or http://localhost:3000 for local dev
export API_KEY="sk_live_your_key_here"
```

## Authentication

All API requests require an `Authorization` header with your API key:

```bash title="Authorization header"
Authorization: Bearer sk_live_your_api_key_here
```

## Verify Connection

Test your API key with the `whoami` endpoint:

```bash title="Verify your API key"
curl -s "$BASE_URL/api/v1/whoami" \
  -H "Authorization: Bearer $API_KEY" | jq .
```

<Callout type="info" title="CLI equivalent">
```bash
hackathon whoami
```
</Callout>

### Response

```json title="response.json"
{
  "tenantId": "org_abc123",
  "keyId": "key_xyz",
  "scopes": ["hackathons:read", "hackathons:write", "webhooks:read", "webhooks:write"]
}
```

## Error Handling

Non-200 responses return JSON with an `error` field and an optional `code`:

```json title="error.json"
{
  "error": "Hackathon not found",
  "code": "not_found"
}
```

Common HTTP status codes:

| Status | Meaning |
|--------|---------|
| `400` | Bad request — invalid or missing fields |
| `401` | Unauthorized — missing or invalid API key |
| `403` | Forbidden — key lacks the required scope |
| `404` | Not found |
| `409` | Conflict — duplicate or invalid state transition |
| `429` | Rate limited — slow down |

## API Namespaces

The API is split across three namespaces with different authentication requirements:

| Namespace | Auth | Description |
|-----------|------|-------------|
| `/api/public/*` | None or Clerk session | Public discovery, participant actions (registration, submissions, voting) |
| `/api/dashboard/*` | API key or Clerk session | Organizer management (hackathon CRUD, judging, prizes, sponsors) |
| `/api/v1/*` | API key only | Programmatic access for jobs and webhooks |

<Callout type="info">
Most organizer and programmatic endpoints accept API keys. Participant-facing actions (registration, submissions, team invitations) require a Clerk session since they act on behalf of a logged-in user.
</Callout>
