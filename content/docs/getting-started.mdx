The Oatmeal platform exposes a REST API. All requests use standard `fetch()` with JSON payloads and API key authentication.

## Get Your API Key

Create an API key from the dashboard at **Settings > API Keys**. Keys start with `sk_live_` and are shown only once.

<Callout type="warn">
Never expose your API key in client-side code. Use environment variables and server-side API routes.
</Callout>

## Verify Connection

<CodeBlock language="typescript" showLineNumbers code={"const res = await fetch(`${process.env.OATMEAL_BASE_URL}/api/v1/whoami`, {\n  headers: {\n    \"Authorization\": `Bearer ${process.env.OATMEAL_API_KEY}`,\n  },\n})\n\nif (!res.ok) {\n  console.error(\"Authentication failed:\", res.status)\n  process.exit(1)\n}\n\nconst data = await res.json()\nconsole.log(\"Connected as tenant:\", data.tenantId)\nconsole.log(\"Key ID:\", data.keyId)\nconsole.log(\"Scopes:\", data.scopes)"}>

## Error Handling

API errors return JSON with an `error` field. Use standard `fetch()` error handling:

<CodeBlock language="typescript" showLineNumbers code={"async function apiCall(path: string, options?: RequestInit) {\n  const res = await fetch(`${process.env.OATMEAL_BASE_URL}${path}`, {\n    ...options,\n    headers: {\n      \"Authorization\": `Bearer ${process.env.OATMEAL_API_KEY}`,\n      \"Content-Type\": \"application/json\",\n      ...options?.headers,\n    },\n  })\n\n  const data = await res.json()\n\n  if (!res.ok) {\n    throw new Error(`API error ${res.status}: ${data.error}`)\n  }\n\n  return data\n}"}>

## API Endpoints

The API is split across two namespaces:

| Namespace | Auth | Description |
|-----------|------|-------------|
| `/api/v1/*` | API key only | Programmatic access for jobs, webhooks |
| `/api/dashboard/*` | API key or session | Schedules, extended job details |
